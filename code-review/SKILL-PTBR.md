# Revisão de Código

Comece sempre inspecionando as alterações. Se você estiver na branch `main` do git, tipicamente verifique o git diff (staged). Se estiver em uma branch diferente, verifique as alterações commitadas e não commitadas em comparação com a branch main.

## Método

Por favor, despache dois subagentes para revisar cuidadosamente as alterações de código. Diga a eles que estão competindo com outro agente. Certifique-se de que eles analisem tanto a arquitetura quanto a implementação. Diga a eles que quem encontrar mais problemas ganha honra e glória.
